# ACME SOC Console

Production-grade Security Operations Center dashboard with multi-case incident management, entity profiling, threat intelligence, and SOAR automation tracking.

## üéØ Features

### Core Capabilities
- **Multi-Case Incident Management** - Track 3+ concurrent security incidents with full case lifecycle
- **Entity Risk Profiling** - User, IP, device, and domain analysis with risk scoring
- **Threat Intelligence** - IOC management (domains, IPs, apps) with tagging and export
- **MITRE ATT&CK Mapping** - Technique coverage across cases with observation counts
- **Playbook Automation** - SOAR playbook execution tracking and simulation
- **Event Timeline** - Real-time security event stream with filtering
- **Command Palette** - Quick navigation and actions via Ctrl+K

### UI/UX Highlights
- **Design Token System** - Consistent spacing, colors, typography via CSS variables
- **Skeleton Loading** - Shimmer animations for perceived performance
- **Micro-Interactions** - Hover lifts, smooth transitions, badge pulses
- **War Room** - Case-specific activity feed with notes (localStorage persistence)
- **Quick Pivot** - Click entity chips to investigate related data
- **Toast Notifications** - Non-intrusive status updates
- **Reduced Motion Support** - Accessibility for motion-sensitive users
- **Dark Mode Optimized** - Professional SOC aesthetic inspired by QRadar/XSOAR

## üöÄ Getting Started

### Local Development

```bash
# 1. Generate data
cd SOC-case-study-project
python run_pipeline.py

# 2. Verify export
ls dashboard/dashboard_data/

# 3. Open dashboard
cd dashboard
# Option A: Direct open (may have CORS issues)
start index.html  # Windows
open index.html   # macOS

# Option B: Local server (recommended)
python -m http.server 8000
# Visit http://localhost:8000
```

### GitHub Pages Deployment

**Automatic (via Actions):**

The workflow `.github/workflows/static.yml` automatically:
1. Runs `python run_pipeline.py` on every push
2. Exports data to `dashboard/dashboard_data/`
3. Deploys `dashboard/` to GitHub Pages

**Manual:**

```bash
# Run pipeline
python run_pipeline.py

# Commit data
git add dashboard/dashboard_data/
git commit -m "Update SOC dashboard data"
git push

# Enable Pages: Settings ‚Üí Pages ‚Üí Source: main branch ‚Üí /dashboard
```

Dashboard will be live at: `https://username.github.io/repo-name/`

## üìñ User Guide

### Navigation

**Sidebar Menu:**
- **Overview** - High-level metrics and recent activity
- **Cases** - Incident case management (primary view)
- **Alerts** - Security alerts grouped by case
- **Entities** - User/IP/device risk profiles
- **Intelligence** - IOC database with export
- **Timeline** - Event stream with filters
- **MITRE ATT&CK** - Technique coverage grid
- **Automations** - Playbook execution logs

**Command Palette (Ctrl+K):**
- Type to filter commands
- Navigate views instantly
- Open cases by ID
- Export data
- Refresh dashboard

### Cases View

**Table Columns:**
- Case ID (clickable)
- Title
- Severity (Critical/High/Medium/Low)
- Status (New/Investigating/Contained/Closed)
- Owner
- Affected users
- Alert count
- Created timestamp

**Filters:**
- Status dropdown
- Severity dropdown

**Click a row** to open the Case Drawer.

### Case Drawer

**Tabs:**
1. **Summary** - Title, narrative, affected assets, timeline
2. **Alerts** - All alerts linked to this case
3. **Entities** - Affected users (clickable to pivot)
4. **MITRE** - Techniques used in this incident

**War Room:**
- View activity feed (status changes, notes, playbook runs)
- Add investigation notes (persisted to browser localStorage)

### Alerts View

Alerts are **grouped by case** for context.

**For each alert:**
- Click row to view in Case Drawer
- Click entity chip to pivot to Entities view
- See evidence count, recommended actions

### Entities View

**Cards show:**
- Email/IP/device identifier
- Risk score with severity badge
- Event count
- Alert count
- Unique IP count

**Click a card** to view detailed profile (future enhancement).

### Intelligence View

**IOC Table:**
- Type (domain, IP, app)
- Indicator value
- Tags (phishing, malicious_oauth, etc.)
- Linked cases
- Copy button

**Export IOCs** button downloads CSV.

### Timeline View

**Event Stream:**
- Most recent 50 events
- Event type, timestamp, description
- Source, user, IP chips
- Color-coded by severity

### MITRE ATT&CK View

**Technique Cards:**
- Technique ID (e.g., T1566.002)
- Technique name
- Tactic (Initial Access, Persistence, etc.)
- Observation count across all cases

### Automations View

**Playbook Run Table:**
- Playbook name
- Linked case ID
- Status (completed/partial)
- Duration
- Actions taken count
- Execution timestamp

## üîß Customization

### Add a New View

1. **HTML:** Add nav item in `index.html`:
```html
<a href="#" class="nav-item" data-view="myview">
    <svg>...</svg>
    <span>My View</span>
</a>
```

2. **JS:** Add render function in `app.js`:
```javascript
function renderMyView(container) {
    container.innerHTML = `
        <h1 class="view-title">My View</h1>
        <!-- content -->
    `;
}

// Add to renderView() switch statement
case 'myview':
    renderMyView(container);
    break;
```

### Modify Color Scheme

Edit `:root` variables in `styles.css`:

```css
:root {
    --bg-primary: #your-color;
    --brand-primary: #your-brand;
}
```

All components use these tokens automatically.

### Add Command Palette Action

In `renderCommandResults()`:

```javascript
commands.push({
    icon: 'üî•',
    label: 'My Action',
    desc: 'Description',
    action: () => { /* your code */ }
});
```

## üêõ Troubleshooting

**"Data Missing" status pill:**
- Run: `python run_pipeline.py`
- Verify: `dashboard/dashboard_data/*.json` exist
- Check: Browser console for fetch errors (F12)

**CORS errors (local file://)**:
- Use local server: `python -m http.server 8000`
- Or deploy to GitHub Pages

**Filters not working:**
- Clear browser cache (Ctrl+Shift+R)
- Check console for JavaScript errors

**Skeleton loaders stuck:**
- Check network tab (F12) for failed requests
- Verify JSON files are valid (not truncated)

**Command palette not opening:**
- Try Cmd+K (macOS) or Ctrl+K (Windows/Linux)
- Check if another extension hijacks this shortcut
