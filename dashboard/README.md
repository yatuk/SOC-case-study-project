# SOC Incident Dashboard

Production-quality static dashboard for visualizing SOC pipeline outputs.

## Overview

This dashboard provides real-time visualization of security incident data from the SOC detection pipeline:

- **KPI Metrics** - Total events, alerts, high-severity counts, unique users
- **Alert Management** - Searchable, filterable alert list with severity badges
- **Evidence Drawer** - Detailed alert information including MITRE ATT&CK mapping
- **Attack Timeline** - Visual incident progression from phishing to containment
- **MITRE Heatmap** - Techniques used in the attack chain
- **Risk Scoring** - User risk score breakdown with contributing factors

## Technology Stack

- **HTML5** - Semantic structure
- **CSS3** - Dark mode, responsive design, animations
- **Vanilla JavaScript** - Zero dependencies, pure ES6+
- **No Backend Required** - Fully static, works on GitHub Pages

## Local Usage

### Option 1: Direct File Access
```bash
# After running the pipeline
cd dashboard
open index.html  # macOS
start index.html  # Windows
xdg-open index.html  # Linux
```

### Option 2: Local Web Server
```bash
# Python 3
python -m http.server 8000

# Node.js
npx http-server

# Then visit: http://localhost:8000
```

## GitHub Pages Deployment

### Automatic Deployment (Recommended)

The repository includes a GitHub Actions workflow that automatically:
1. Runs the SOC pipeline on every push to `main`
2. Generates fresh incident data
3. Deploys the dashboard to GitHub Pages

**Setup:**
1. Go to repository Settings → Pages
2. Set Source to "GitHub Actions"
3. Push to `main` branch
4. Dashboard will be live at: `https://yourusername.github.io/SOC-case-study-project/`

### Manual Deployment

```bash
# Run pipeline locally
python run_pipeline.py

# Push to GitHub
git add .
git commit -m "Update dashboard data"
git push origin main
```

## Data Sources

The dashboard loads data from `dashboard_data/`:

| File | Content |
|------|---------|
| `summary.json` | Incident metadata, KPIs, timeline |
| `alerts.jsonl` | Detection alerts (JSON Lines format) |
| `risk_scores.json` | User risk scores and contributing factors |
| `correlations.json` | Event correlation patterns |

## Features

### Alert List
- **Search** - Filter by user, IP, alert name
- **Severity Filter** - Critical, High, Medium, Low
- **Click to Expand** - View full alert details in drawer

### Alert Details Panel
- Alert name, severity, confidence
- Affected entity (click to view risk score)
- Time window
- Security hypothesis
- MITRE ATT&CK techniques with tactics
- Evidence event IDs (up to 10 shown)
- Recommended remediation actions
- Copy-to-clipboard for actions

### Attack Timeline
- Visual flow of incident progression
- Color-coded severity markers
- Timestamps and descriptions
- Hover animations

### MITRE ATT&CK Heatmap
- All detected techniques
- Active techniques highlighted in cyan
- Tactic categorization

### Risk Score Modal
- Total user risk score
- Severity classification
- Individual contributing factors with point values
- Textual explanation of scoring methodology

## Troubleshooting

**Dashboard shows "Failed to load data"**
- Ensure pipeline has run: `python run_pipeline.py`
- Check `dashboard_data/` exists and contains JSON files
- If using file://, some browsers block local AJAX - use a web server

**Alerts not appearing**
- Verify `alerts.jsonl` is valid JSON Lines format
- Check browser console for errors
- Ensure severity filter is set to "All"

**GitHub Pages 404**
- Verify Actions workflow completed successfully
- Check Settings → Pages shows deployment URL
- May take 1-2 minutes after push to go live
